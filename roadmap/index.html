Project roadmap

From retrofit beta to open infrastructure.

OpenParcelBox starts with a concrete standalone retrofit for existing DHL parcel boxes, then grows into an open ecosystem for locks, bridges, integrations and community parcel infrastructure.

Milestones

Near-term build order

M1: DHL Retrofit Beta

Standalone ESP-based retrofit target for existing DHL parcel boxes with local users, carrier PINs, NFC, Wiegand and local event logging.

 Firmware platform decision

Decide whether the first firmware foundation should use Arduino Core, ESP-IDF, ESPHome, a custom stack or a hybrid path.

 Lock retrofit strategy

Explore whether the old lock unit is removed, gutted, reused or replaced by a simpler OpenParcelLock-style mechanism.

 Standalone power budget

Estimate solar, battery, sleep behavior, reader current and lock actuation power before hardware assumptions harden.

Next practical step

Document the real parts.

The next roadmap work starts with photos, measurements and component notes: parcel box, lock area, keypad/RFID module, ESP board, sensors, cable paths and power modules.

Later platform work

Bridge, cloud and carrier APIs come later.

M1 must work locally without cloud, bridge, Home Assistant or internet. Those integrations are important, but they should build on a reliable local foundation.

OpenParcelBox is open source and local-first. The optional cloud is convenience, not a requirement.
