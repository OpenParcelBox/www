Why it exists

Parcel boxes should not die with a closed access platform.

OpenParcelBox starts from the opposite assumption: the box remains useful even if a service provider discontinues its closed access system. The lock is replaceable, the rules are auditable and the owner keeps local control.

March 2025 The trigger

Project origin

The official fix was a combination lock. That says a lot.

When DHL announced that electronic access to older DHL parcel boxes would end on 15 May 2025, the hardware did not suddenly become useless. The closed access system did. The pragmatic official advice was almost brutally simple: make the box manual again, attach a code lock or latch, and store the code in the delivery instructions for the alternative drop-off location.

The funny part

That workaround accidentally turns a formerly DHL-only smart box into something every carrier can understand: open this receptacle with this right, under these conditions.

The serious part

A handwritten code is not a platform. OpenParcelBox is the attempt to make that idea safe, auditable, local-first and ready for dynamic carrier-neutral access.

Local first

The device must keep its basic local function without internet, cloud services or Home Assistant.

Carrier neutral

Carrier integrations should be optional plugins, never a single-vendor dependency or hidden lock-in path.

Retrofit friendly

Existing parcel boxes, cabinets, lockers and garage boxes should remain useful instead of being replaced by a closed product.

For manufacturers

Sell the box. Do not trap the user.

Commercial products are welcome. Ready-to-use retrofit kits, locks and parcel boxes can absolutely exist. But official OpenParcelBox compatibility should mean repairable, documented and interoperable hardware, not another dead end when a service platform, spare part or private protocol disappears.

Access flow

One right. Many ways to present it.

OpenParcelBox models access as an opening right that can become a PIN, QR code, signed token, app action or future carrier grant.

01Issue

Create a scoped deposit-only right.

02Present

Render it as PIN, QR or token.

03Validate

Check locally, even offline.

04Audit

Log the event without raw secrets.

Standards-oriented

Built around public interoperability goals.

OpenParcelBox uses CEN/TS 17457:2020 as an important reference for interoperable digital opening and closing systems for home-use parcel receptacles. Public documentation avoids copying protected standards text.

Optional integrations

Smart home is welcome, never required.

MQTT, Home Assistant, HomeMatic, Matter, webhooks, LoRa/LoRaWAN and cloud services are integration paths, not prerequisites for local operation.
